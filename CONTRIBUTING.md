# Issue and Contribution Guidelines / 이슈·기여 원칙

This repository distributes proprietary binaries and public documentation. It is not the
application source repository and does not accept source-code pull requests for PCssak Palette.
Documentation corrections and well-reproduced bug reports are welcome through GitHub Issues.

## Responsible reports

- Search existing issues before opening a new one.
- Reproduce the behavior on the latest release when possible.
- Describe the user problem, expected result, actual result, and minimal steps.
- Remove personal data, secrets, customer files, and confidential designs.
- Use the private security route for exploitable details.
- Remain available to answer reasonable clarification questions.

AI may help prepare a report, but the submitter must verify every factual claim and reproduction
step. Do not submit bulk-generated, speculative, duplicate, or untested issues. PCSSAK applies
the same rule when reporting to upstream open-source maintainers.

## 한국어

이 저장소는 독점 바이너리와 공개 문서를 배포하며 애플리케이션 소스 저장소가 아닙니다.
PCssak Palette 소스 코드 Pull Request는 받지 않지만, 문서 오류와 직접 재현한 버그 제보는
GitHub Issue로 받을 수 있습니다.

기존 이슈를 먼저 검색하고 가능한 경우 최신 버전에서 확인한 뒤, 사용자 문제·기대 결과·실제
결과·최소 재현 절차를 적어주세요. 개인정보·비밀정보·고객 파일·회사 디자인은 제거하고,
악용 가능한 보안 내용은 비공개 경로를 사용하세요.

AI로 제보 작성을 도울 수 있지만 모든 사실과 재현 단계는 제출자가 직접 검증해야 합니다.
추측성·중복·미검증 이슈를 대량으로 등록하지 마세요. PCSSAK도 다른 오픈소스 관리자에게
같은 원칙을 지킵니다.
