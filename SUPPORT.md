# Support / 고객지원

PCssak Palette is maintained by a solo developer. Reports are prioritized by user impact,
security, data-loss risk, and reproducibility rather than arrival order.

## Before reporting

1. Confirm the app version and Windows architecture.
2. Check the [latest release notes](https://github.com/pcssakinc/pcssak-palette-releases/releases/latest)
   and [known limitations](docs/KNOWN-LIMITATIONS.md).
3. Reproduce the problem with the smallest safe example.
4. Remove personal, customer, company-confidential, or licensed source material from screenshots
   and attachments.

Use the GitHub [bug form](../../issues/new?template=bug-report.yml) for reproducible application
defects and the [feature form](../../issues/new?template=feature-request.yml) for a user problem
that the product could solve. General help is also available at `pcssakinc@gmail.com`.

Security issues that could be exploited must follow [SECURITY.md](SECURITY.md) and should not be
posted publicly.

## 한국어

PCssak Palette는 1인 개발자가 운영합니다. 문의는 도착 순서만이 아니라 사용자 영향, 보안,
데이터 손실 위험과 재현 가능성을 기준으로 우선순위를 정합니다.

제보 전에 앱 버전과 Windows 아키텍처를 확인하고, 최신 릴리스 노트와
[알려진 한계](docs/KNOWN-LIMITATIONS.ko.md)를 살펴본 뒤 가장 작은 안전한 예제로 재현해 주세요.
스크린샷과 첨부물에서는 개인정보, 고객 자료, 회사 비밀과 라이선스가 있는 원본 자료를
제거하세요.

재현 가능한 오류는 GitHub 버그 양식, 해결이 필요한 사용자 문제는 기능 제안 양식을 사용합니다.
일반 문의는 `pcssakinc@gmail.com`으로 보낼 수 있습니다. 악용 가능한 보안 문제는 공개하지
말고 `SECURITY.md` 절차를 따르세요.
