# 품질과 안전

[English](QUALITY-AND-SAFETY.md)

이 문서는 PCSSAK이 현재 위험을 줄이기 위해 하는 일을 설명합니다. 오류가 전혀 없다는 약속이
아니라 개발·검증 과정의 근거입니다.

## 로컬 우선 경계

- 이미지·색·팔레트·검증·내보내기는 기기에서 처리합니다.
- 공개 베타에는 PCSSAK 계정, 광고, 사용 분석, 추적, 자동 오류 업로드와 사용자 데이터베이스가
  없습니다.
- 핵심 색상 작업은 오프라인에서도 사용할 수 있습니다.
- 독립형 베타는 공식 GitHub의 서명된 업데이트 확인·다운로드에만 연결되며, 업데이트 확인에
  실패해도 색상 작업은 막히지 않습니다.

## 제한된 네이티브 입력

Rust 경계에서 신뢰할 수 없는 이미지를 전체 처리하기 전에 검사합니다.

- 파일 최대 40 MiB
- 최대 4,000만 픽셀
- 디코더 할당 최대 256 MiB
- 분석 전 긴 변을 최대 128픽셀로 축소
- 지원 형식 PNG·JPEG·WebP·GIF·BMP

가능한 경우 전체 디코딩 전에 메타데이터를 검사합니다. 투명 픽셀은 축소 전에 처리해 어두운
테두리 오염을 줄이고, 고정 시드 CIELab k-means로 같은 입력과 빌드에서 결과를 재현합니다.

## 최소 권한과 출력 검증

WebView에 광범위한 파일시스템 권한을 주지 않고 대화상자, 텍스트 클립보드 쓰기, 업데이트와
재시작 권한만 좁게 제공합니다. 사용자가 고른 파일은 전용 Rust 명령으로 처리합니다.

Free CSS는 `.css`, 1 MiB 이하, PCssak 생성 표시와 `:root`, NUL 바이트 없음 조건을 다시
검사합니다. 라이브러리 백업은 `.json`, 4 MiB 이하, 올바른 JSON과 에디션별 저장 한도를
검증합니다.

## 객관적 계산

- OKLCH 램프, WCAG 비율, 보정 제안과 내보내기는 결정론적입니다.
- WCAG는 표시된 색 조합의 상대 휘도를 계산합니다.
- 색각이상 근사 보기는 선형 RGB의 Machado 행렬을 사용합니다.
- PCSSAK 색 거리 기준은 검토 휴리스틱이며 WCAG 기준과 구분해 표시합니다.
- 주관적인 아름다움·전문성 점수는 제공하지 않습니다.

## 공개 Free 보호

공개 빌드는 Free가 기본입니다. 내부 Pro는 컴파일 기능과 별도 로컬 확인이 필요하고 Store 예약
기능과 함께 빌드할 수 없습니다. Pro 전용 네이티브 작업은 권한을 다시 검사합니다. 공개 검증기는
내부 플래그, 의심스러운 키 파일, 버전 불일치와 승인되지 않은 업데이트 주소를 거부합니다.

## 배포 무결성

- 앱·Tauri·Rust 패키지·잠금 파일·UI 버전의 일치를 확인합니다.
- 의존성을 잠그고 상업적 라이선스 정책에 맞는지 검사합니다.
- 전체 제3자 고지문을 생성해 앱에 포함합니다.
- x64·x86 업데이트 패키지에 아키텍처별 Tauri 서명을 적용합니다.
- `latest.json`, 분리된 `.sig`, `SHA256SUMS.txt`를 릴리스에 포함합니다.
- 배포 후 공개 파일 크기와 해시를 테스트한 로컬 파일과 대조합니다.
- 공개 스크립트는 명시적 확인이 필요하며 커밋되지 않은 소스에서는 중단합니다.

## 검증 계층

색상 엔진, 권한, 실행 취소, 저장, 업데이트, 번역 완전성, Rust 입력 경계 테스트와 TypeScript
프로덕션 빌드, Rust 포맷·Clippy, 라이선스 검사, 설치 앱 실행, 중복 실행, 업데이트·업그레이드,
좁은 화면과 데스크톱 시각 QA를 사용합니다. 정확한 결과는 영구적인 숫자로 광고하지 않고 날짜가
있는 릴리스 노트와 업무일지에 연결합니다.

## 사용자도 확인해야 하는 부분

- 공식 PCSSAK 경로에서만 내려받으세요.
- 최초 미서명 설치 파일은 이름과 SHA-256을 비교하세요.
- SmartScreen이나 Defender를 끄지 마세요.
- 중요한 CSS와 JSON 백업은 별도로 보관하세요.
- 색각 근사 보기와 WCAG 결과를 의료·법적 인증으로 사용하지 마세요.
- 재현 가능한 오류와 보안 문제는 정해진 경로로 알려주세요.
