# PCssak Palette를 만든 이유

[English](PROJECT-STORY.md)

## 믿기 어려웠던 색에서 시작했습니다

PCssak Palette는 기능 목록이 아니라 기획자의 실제 불편에서 시작했습니다. 기획자는 청록색과
녹색을 구분하기 어려운 색각이상이 있는 한국의 1인 개발자입니다. 같은 색을 다시 찾거나, 두
색이 정말 다른지 판단하거나, 상태 색을 선택할 때 추측하거나 다른 사람의 확인을 받아야 하는
경우가 있었습니다.

처음 생각한 것은 색을 더 안정적으로 찾고 비교하는 작은 도구였습니다. 이후 질문이 커졌습니다.

> 색각이상이 있는 사람도 소프트웨어가 아름다움을 대신 판단한다고 주장하지 않으면서, 객관적인
> 근거를 사용해 전문적인 색상 시스템을 만들 수 있을까?

그래서 PCssak Palette는 대비 비율, 색각이상 근사 화면에서의 구별 가능성, 역할 색 충돌,
결정론적 보정과 안정적인 납품 형식처럼 반복해서 설명할 수 있는 질문에 집중합니다. 취향, 문화,
브랜드 의미와 최종 디자인 판단은 사람에게 남겨 둡니다.

## 개발 과정

아래 내용은 기획자가 사용한 도구의 기록이며 해당 회사의 후원·보증이나 책임 이전을 뜻하지
않습니다.

- **2026년 1월:** Antigravity로 학습과 프로토타입을 시작하고 여러 PC 프로그램 아이디어를
  제품 계획으로 구체화했습니다.
- **기획 단계:** Claude Opus 4.5의 도움으로 초기 제품 구조와 요구사항을 발전시켰습니다.
- **현재:** OpenAI Codex의 도움으로 구현 검토, 회귀 작업, 배포 공학과 문서화를 진행합니다.

AI는 속도를 높였지만 제품의 주인은 아닙니다. 해결할 문제, 사용자가 감수할 선택, 계정과 키,
출시 결정과 유지보수는 기획자가 관리합니다. AI를 품질 보증처럼 내세우는 대신
[AI 개발 정책](AI-DEVELOPMENT-POLICY.ko.md)을 공개하는 이유입니다.

## PCSSAK이 얻고 싶은 신뢰

PCSSAK은 집중된 Windows 도구들을 만들어 가고 있습니다. 신뢰는 다음처럼 확인할 수 있는
행동에서 얻어야 한다고 생각합니다.

- 기능 범위와 알려진 한계를 명확하게 공개하기
- 가능한 작업은 기기에서 처리하고 권한을 좁게 유지하기
- 객관적 주장에는 테스트와 결정론적인 계산을 사용하기
- 의존성과 라이선스를 검토하고 고지문을 포함하기
- 업데이트 서명, 해시와 배포 기록을 남기기
- Early Access라는 말 뒤에 숨지 않고 버그를 기록하고 고치기

PCssak Palette는 PCSSAK의 첫 공개 제품입니다. 큰 회사처럼 보이는 것이 목표가 아닙니다.
한국의 작은 소프트웨어 프로젝트도 꼼꼼하고 이해 가능하며 계속 믿을 만하다는 것을 행동으로
보여주는 것이 목표입니다.
