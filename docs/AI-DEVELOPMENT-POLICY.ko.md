# AI 보조 개발 정책

[English](AI-DEVELOPMENT-POLICY.md)

PCSSAK은 조사·구현·검토·테스트·번역·문서화에 AI 도구를 사용합니다. AI의 결과는 제안이며
정답의 증거나 출시 결정이 아닙니다.

## 사람의 책임

1인 개발자는 요구사항, 제품 범위, 계정과 키, 위험 수용, 출시 승인, 사용자 안내와 유지보수를
관리합니다. AI 제공사는 PCssak Palette의 공동 소유자, 보안 연락처 또는 배포자가 아닙니다.

## 변경을 배포할 수 있는 조건

1. 사용자 문제와 완료 조건이 분명해야 합니다.
2. 현재 구조를 따르고 설명되지 않은 별도 경로를 만들지 않아야 합니다.
3. 데이터 흐름, 중요한 가정, 실패 형태와 되돌리는 방법을 설명할 수 있어야 합니다.
4. 위험에 맞는 테스트나 반복 가능한 확인 결과가 있어야 합니다.
5. 새 의존성과 API의 출처·관리 상태·버전·상업적 라이선스를 확인해야 합니다.
6. 관련된 파일 입력, 권한, 네트워크, 개인정보, 업데이트 무결성과 자원 고갈 경계를 검토합니다.
7. 지원 Windows, 번역, 업데이트와 사용자 복구를 고려합니다.
8. 동작이 바뀌면 사용자 문서와 알려진 한계도 함께 바꿉니다.
9. 공개 배포, 서명, Store와 유료 권한 작업은 기획자가 명시적으로 승인합니다.

컴파일만 되는 것은 증거가 아닙니다. 생성된 구현을 그대로 반복하는 생성 테스트 역시 독립적인
증거가 아닙니다.

## 의도적으로 하지 않는 일

- 검토하지 않은 생성 코드를 배포하기
- 빌드를 통과시키려고 테스트를 지우거나 약화하기
- 실제 존재와 라이선스를 확인하지 않은 패키지 추가하기
- 완벽한 보안·인증·의료 정확성·미지원 호환성을 주장하기
- 중요한 네트워크 연결·수집·한계를 숨기기
- 비밀 키, 서명 키, 사용자 비공개 파일이나 회사 자료를 공개하기
- 검증하지 않은 추측성 이슈와 패치를 오픈소스 관리자에게 대량 등록하기
- 소스 라이선스 없이 공개 바이너리 저장소를 오픈소스라고 설명하기

## 실수를 발견했을 때

영향 버전과 재현 방법을 기록하고, 원인을 찾고, 가능한 경우 회귀 검사를 추가한 뒤 유지 가능한
수정을 적용합니다. 위험에 맞는 배포 검증을 다시 실행하고 남은 위험과 복구 방법을 기록합니다.
가능하면 같은 실수가 반복되지 않도록 테스트·검증기·체크리스트나 설계 규칙으로 남깁니다.
